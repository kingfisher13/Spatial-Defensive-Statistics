About this Project

This is a project by Josh Ryan. It aims to explore the relationship between defense and space as played in the NBA (National Basketball Association). The algorithms used to generate the various statistics are described below. The Visualization page shows an example game with the currently selected algorithm. Blue lines are the defensive pairings (who is guarding who) that are used to calculate the Plus/Minus statistics in the table.

Simple Closest

For each frame, the algorithm picks a random player and determines the closest opposing team member and pairs the two. Then it picks another random player, checks to see if the player has been picked or paired before, and then if not, finds the closest opposing team member and pairs it.

Downsides:

  • Multiple defenders can be penalized for a basket. Sometimes this is appropriate, others times, not so much (e.g. big man rotates to crash the boards and is paired with the shooter milliseconds before the shot).
  • Great offensive players can "hide" their defensive liabilties by scoring a lot. See James Harden.
  • Really bad defenders who are completely blown past are not necessarily penalized if a help defender ends up being closer at the moment of the shot.
  • Simple Closest does not do any analysis on shots that don't go in. This obviously is bad from a measuring defensive ability standpoint.
